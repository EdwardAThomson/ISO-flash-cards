# Technical Security Controls

## Operations Security

Technical controls protect systems, networks, and data from threats through technology-based measures.

## Annex A: Endpoint & Configuration (A.8.1, A.8.9)

### A.8.1 User Endpoint Devices

**Control:** Information on user endpoint devices shall be protected.

**Endpoint types:**
- Laptops and desktops
- Mobile phones and tablets
- USB devices
- Wearables

**Protection measures:**
- Encryption (full disk, file-level)
- Endpoint protection (antivirus, EDR)
- Mobile device management (MDM)
- Remote wipe capability
- Patch management

### A.8.9 Configuration Management

**Control:** Configurations shall be established, documented, implemented, monitored, and reviewed.

**Configuration management includes:**
- Baseline configurations
- Hardening standards
- Change control
- Configuration monitoring
- Drift detection

## Annex A: Malware & Vulnerabilities (A.8.7–A.8.8)

### A.8.7 Protection Against Malware

**Control:** Protection against malware shall be implemented.

**Malware types:**

| Type | Description |
|------|-------------|
| **Virus** | Self-replicating, attaches to files |
| **Worm** | Self-replicating, spreads via network |
| **Trojan** | Disguised as legitimate software |
| **Ransomware** | Encrypts data, demands payment |
| **Spyware** | Collects information covertly |
| **Rootkit** | Hides malicious activity |

**Protection measures:**
- Antivirus/anti-malware software
- Email filtering
- Web filtering
- Application whitelisting
- User awareness training
- Sandboxing

## Annex A: Cryptography (A.8.24)

### A.8.24 Use of Cryptography

**Control:** Rules for effective use of cryptography shall be defined and implemented.

### Cryptographic Concepts

| Concept | Purpose |
|---------|---------|
| **Encryption** | Protect confidentiality |
| **Hashing** | Verify integrity |
| **Digital signatures** | Verify authenticity and non-repudiation |
| **Key management** | Secure generation, storage, distribution |

### Encryption Types

| Type | Use Case |
|------|----------|
| **Symmetric** | Fast, bulk data encryption (AES) |
| **Asymmetric** | Key exchange, digital signatures (RSA, ECC) |
| **Hybrid** | Combines both (TLS) |

### Key Management

| Activity | Considerations |
|----------|----------------|
| **Generation** | Sufficient randomness, appropriate length |
| **Storage** | Hardware security modules, key vaults |
| **Distribution** | Secure channels, out-of-band verification |
| **Rotation** | Regular replacement, crypto-period limits |
| **Destruction** | Secure deletion, key escrow |

## Annex A: Network Security (A.8.20–A.8.23)

### A.8.20 Networks Security

**Control:** Networks shall be managed and controlled.

**Network security measures:**
- Firewalls
- Intrusion detection/prevention (IDS/IPS)
- Network segmentation
- VPNs
- Network access control (NAC)

### A.8.21 Security of Network Services

**Control:** Security mechanisms and service levels shall be identified and included in agreements.

### A.8.22 Segregation of Networks

**Control:** Groups of services, users, and systems shall be segregated.

**Segmentation approaches:**
- VLANs
- Subnets
- Firewalls between zones
- Micro-segmentation
- Zero trust architecture

### A.8.23 Web Filtering

**Control:** Access to external websites shall be managed.

**Filtering categories:**
- Malware sites
- Phishing sites
- Inappropriate content
- High-risk categories
- Data exfiltration channels

## Annex A: Logging & Monitoring (A.8.15–A.8.17)

### A.8.15 Logging

**Control:** Logs shall be produced, stored, protected, and analyzed.

**Log types:**

| Log Type | Content |
|----------|---------|
| **Security logs** | Authentication, authorization, security events |
| **System logs** | OS events, errors, performance |
| **Application logs** | Application-specific events |
| **Network logs** | Traffic, connections, firewall events |
| **Audit logs** | Administrative actions, configuration changes |

**Log management:**
- Centralized collection (SIEM)
- Secure storage
- Retention policies
- Integrity protection
- Regular review

### A.8.16 Monitoring Activities

**Control:** Networks, systems, and applications shall be monitored.

**Monitoring approaches:**
- Real-time alerting
- Anomaly detection
- Behavioral analysis
- Threat hunting
- Security operations center (SOC)

### A.8.17 Clock Synchronization

**Control:** Clocks shall be synchronized to approved time sources.

**Importance:**
- Accurate log correlation
- Forensic analysis
- Regulatory compliance
- Certificate validation

## Annex A: Secure Development (A.8.25–A.8.31)

### A.8.25 Secure Development Lifecycle

**Control:** Rules for secure development shall be established.

### A.8.26 Application Security Requirements

**Control:** Security requirements shall be identified and specified.

### A.8.27 Secure System Architecture and Engineering

**Control:** Principles for secure systems shall be established and applied.

### A.8.28 Secure Coding

**Control:** Secure coding principles shall be applied.

**Secure coding practices:**
- Input validation
- Output encoding
- Parameterized queries
- Error handling
- Secure authentication
- Session management

### A.8.29 Security Testing in Development and Acceptance

**Control:** Security testing shall be defined and implemented.

**Testing types:**
- Static analysis (SAST)
- Dynamic analysis (DAST)
- Penetration testing
- Code review
- Fuzz testing

### A.8.30 Outsourced Development

**Control:** Security requirements for outsourced development shall be addressed.

### A.8.31 Separation of Development, Test and Production

**Control:** Environments shall be separated.

## Annex A: Data Protection (A.8.10–A.8.14)

### A.8.10 Information Deletion

**Control:** Information shall be deleted when no longer required.

### A.8.11 Data Masking

**Control:** Data masking shall be used in accordance with policies.

**Masking techniques:**
- Substitution
- Shuffling
- Encryption
- Nulling
- Tokenization

### A.8.12 Data Leakage Prevention

**Control:** DLP measures shall be applied.

**DLP controls:**
- Content inspection
- Endpoint monitoring
- Network monitoring
- Cloud access security brokers (CASB)
- Email filtering

### A.8.13 Information Backup

**Control:** Backup copies shall be maintained and tested.

**Backup considerations:**
- Frequency (RPO)
- Retention
- Offsite storage
- Encryption
- Regular testing

### A.8.14 Redundancy of Information Processing Facilities

**Control:** Facilities shall have sufficient redundancy.

---

## Key Takeaways

1. **Defense in depth** — Multiple layers of controls
2. **Patch promptly** — Vulnerability management is critical
3. **Encrypt sensitive data** — At rest and in transit
4. **Segment networks** — Limit lateral movement
5. **Log everything** — Detection depends on visibility
6. **Secure development** — Build security in from the start

# A5: Data & Model Governance

## Data for AI Systems (Annex A.7)

Data is fundamental to AI systems. ISO 42001 includes controls to ensure data is managed appropriately throughout the AI lifecycle.

## Data Management Controls

### A.7.2 Data Acquisition

**Control:** Processes for acquiring data for AI systems shall be established.

**Implementation considerations:**

| Aspect | Requirements |
|--------|--------------|
| **Source identification** | Document where data comes from |
| **Legal basis** | Ensure lawful acquisition (consent, contract, legitimate interest) |
| **Quality assessment** | Evaluate data quality before use |
| **Provenance** | Track data origin and chain of custody |
| **Third-party data** | Assess vendor data practices |

### A.7.3 Data Quality

**Control:** Data quality for AI systems shall be managed.

**Quality dimensions:**

| Dimension | Description |
|-----------|-------------|
| **Accuracy** | Data correctly represents reality |
| **Completeness** | No missing values or gaps |
| **Consistency** | No contradictions within or across datasets |
| **Timeliness** | Data is current and relevant |
| **Relevance** | Data is appropriate for intended use |
| **Representativeness** | Data reflects the target population |

**Quality management activities:**
- Define quality requirements
- Measure quality metrics
- Identify and address quality issues
- Document quality assessments

### A.7.4 Data Preparation

**Control:** Data preparation for AI systems shall be documented.

**Documentation should cover:**
- Data cleaning procedures
- Transformation and normalization
- Feature engineering
- Handling of missing values
- Outlier treatment
- Data augmentation techniques

### A.7.5 Data Provenance

**Control:** Data provenance for AI systems shall be recorded.

**Provenance information:**
- Original data source
- Collection date and method
- Transformations applied
- Personnel involved
- Quality assessments performed
- Version history

## Bias and Fairness in Data

### Identifying Data Bias

| Bias Type | Description | Example |
|-----------|-------------|---------|
| **Selection bias** | Non-representative sampling | Training on data from one region only |
| **Historical bias** | Past discrimination in data | Hiring data reflecting past biases |
| **Measurement bias** | Inconsistent data collection | Different criteria for different groups |
| **Label bias** | Biased labeling decisions | Subjective annotations |
| **Aggregation bias** | Inappropriate grouping | Ignoring subgroup differences |

### Mitigating Data Bias

1. **Assess representativeness** — Compare data to target population
2. **Identify protected attributes** — Understand sensitive characteristics
3. **Measure disparities** — Analyze outcomes across groups
4. **Apply corrections** — Resampling, reweighting, or augmentation
5. **Document limitations** — Acknowledge remaining biases

## Model Development Controls

### A.6.2.4 Design and Development Documentation

**Control:** Design and development of AI systems shall be documented.

**Documentation requirements:**

| Element | Content |
|---------|---------|
| **Architecture** | Model structure, components, interfaces |
| **Algorithms** | Methods used, rationale for selection |
| **Training process** | Hyperparameters, optimization approach |
| **Assumptions** | Underlying assumptions and constraints |
| **Dependencies** | Libraries, frameworks, external services |

### A.6.2.5 AI System Verification and Validation

**Control:** AI systems shall be verified and validated.

**Verification activities:**
- Code reviews
- Unit testing
- Integration testing
- Static analysis

**Validation activities:**
- Performance testing on held-out data
- Fairness and bias testing
- Robustness testing
- User acceptance testing

### A.6.2.6 AI System Testing

**Control:** AI systems shall be tested.

**Testing considerations:**

| Test Type | Purpose |
|-----------|---------|
| **Functional testing** | System performs intended functions |
| **Performance testing** | Meets accuracy and efficiency requirements |
| **Fairness testing** | No discriminatory outcomes |
| **Robustness testing** | Handles edge cases and adversarial inputs |
| **Security testing** | Resistant to attacks |
| **Integration testing** | Works with other systems |

## Model Documentation

### Model Cards

Document key information about AI models:

| Section | Content |
|---------|---------|
| **Model details** | Name, version, type, architecture |
| **Intended use** | Primary use cases, users, context |
| **Training data** | Data sources, size, characteristics |
| **Evaluation data** | Test datasets, evaluation methodology |
| **Performance metrics** | Accuracy, precision, recall, etc. |
| **Limitations** | Known weaknesses, failure modes |
| **Ethical considerations** | Bias, fairness, privacy implications |

### Version Control

- Track model versions
- Document changes between versions
- Maintain ability to reproduce previous versions
- Link models to training data and code

## Data Protection and Privacy

### Privacy Considerations

| Principle | Implementation |
|-----------|----------------|
| **Data minimization** | Collect only necessary data |
| **Purpose limitation** | Use data only for stated purposes |
| **Storage limitation** | Retain data only as long as needed |
| **Accuracy** | Keep data accurate and up-to-date |
| **Security** | Protect data from unauthorized access |
| **Accountability** | Demonstrate compliance |

### Privacy-Enhancing Techniques

| Technique | Description |
|-----------|-------------|
| **Anonymization** | Remove identifying information |
| **Pseudonymization** | Replace identifiers with pseudonyms |
| **Differential privacy** | Add noise to protect individuals |
| **Federated learning** | Train without centralizing data |
| **Synthetic data** | Generate artificial data for training |

|------|------------------|
| **Data Owner** | Accountable for data assets |

## Key Takeaways

1. **Data quality is critical** — Poor data leads to poor AI
2. **Bias must be addressed** — Identify and mitigate data biases
3. **Documentation enables accountability** — Track data and model decisions
4. **Privacy by design** — Build privacy protection into processes
5. **Governance provides structure** — Clear roles and responsibilities
6. **Version control** — Track changes to data and models

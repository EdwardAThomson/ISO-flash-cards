# Performance & Improvement

## Performance Evaluation (Clause 9)

Clause 9 ensures the ISMS is working effectively through monitoring, measurement, audit, and management review.

## Monitoring, Measurement, Analysis and Evaluation (Clause 9.1)

### What to Evaluate

Organizations must determine:
- **What** needs to be monitored and measured
- **Methods** for monitoring, measurement, analysis, and evaluation
- **When** monitoring and measuring shall be performed
- **Who** shall monitor and measure
- **When** results shall be analyzed and evaluated
- **Who** shall analyze and evaluate

### ISMS Metrics

| Category | Example Metrics |
|----------|-----------------|
| **Policy compliance** | % of staff completing awareness training |
| **Control effectiveness** | % of systems with current patches |
| **Risk management** | Number of high risks treated |
| **Incident management** | Mean time to detect/respond |
| **Audit performance** | Number of findings, closure rate |

### Security Metrics

| Metric Type | Examples |
|-------------|----------|
| **Leading** | Training completion, vulnerability scan coverage |
| **Lagging** | Incidents, breaches, audit findings |
| **Operational** | Patch compliance, access review completion |
| **Strategic** | Risk reduction, maturity improvement |

### Metrics Best Practices

- Align with objectives
- Make actionable
- Keep manageable (not too many)
- Automate collection where possible
- Review and refine regularly
- Present clearly to stakeholders

## Internal Audit (Clause 9.2)

### Audit Requirements

Organizations must conduct internal audits at planned intervals to determine if the ISMS:
- Conforms to organization's own requirements
- Conforms to ISO 27001 requirements
- Is effectively implemented and maintained

### Audit Program

| Element | Consideration |
|---------|---------------|
| **Frequency** | Based on importance and previous results |
| **Scope** | What will be audited |
| **Criteria** | Against what standards |
| **Methods** | Interviews, document review, observation |
| **Auditors** | Competent, objective, impartial |
| **Reporting** | To relevant management |

### Audit Findings

| Type | Description | Action |
|------|-------------|--------|
| **Major NC** | Significant failure to meet requirements | Immediate corrective action |
| **Minor NC** | Isolated or minor failure | Corrective action required |
| **Observation** | Potential improvement area | Consider for improvement |
| **Positive** | Good practice | Recognize and share |

## Management Review (Clause 9.3)

### Review Requirements

Top management must review the ISMS at planned intervals to ensure:
- Continuing suitability
- Adequacy
- Effectiveness

### Review Inputs (9.3.2)

| Input | Content |
|-------|---------|
| **Previous actions** | Status of actions from prior reviews |
| **Changes** | External/internal issues, stakeholder needs |
| **Performance** | Nonconformities, monitoring results, audit results, objectives |
| **Feedback** | From interested parties |
| **Risk assessment** | Results and status of risk treatment |
| **Improvement** | Opportunities for improvement |

### Review Outputs (9.3.3)

| Output | Content |
|--------|---------|
| **Improvement** | Decisions on improvement opportunities |
| **Changes** | Any changes needed to the ISMS |
| **Resources** | Resource needs |

### Review Best Practices

- Schedule regularly (at least annually)
- Use structured agenda
- Prepare materials in advance
- Ensure top management participation
- Document decisions and actions
- Track action completion

## Improvement (Clause 10)

### Continual Improvement (10.1)

Organizations must continually improve the suitability, adequacy, and effectiveness of the ISMS.

### Improvement Sources

| Source | Examples |
|--------|----------|
| **Audit findings** | Nonconformities and observations |
| **Management review** | Decisions and recommendations |
| **Performance data** | Metrics indicating issues |
| **Incidents** | Lessons learned |
| **Risk changes** | New or changed risks |
| **Technology** | New capabilities |
| **Regulations** | New requirements |
| **Feedback** | Stakeholder input |

### Nonconformity and Corrective Action (10.2)

When a nonconformity occurs:

1. **React** — Control and correct it, deal with consequences
2. **Evaluate** — Determine if action needed to eliminate cause
3. **Implement** — Take corrective action
4. **Review** — Assess effectiveness
5. **Update** — Modify ISMS if necessary

### Root Cause Analysis

| Technique | Description |
|-----------|-------------|
| **5 Whys** | Ask "why" repeatedly |
| **Fishbone** | Categorize potential causes |
| **Fault tree** | Map causal relationships |
| **Pareto** | Identify most significant causes |

### Corrective Action Requirements

Corrective actions must be:
- Appropriate to the effects
- Address root cause
- Prevent recurrence
- Documented
- Verified for effectiveness

## Certification Process

### Certification Journey

```
┌─────────────────────────────────────────────────────────┐
│              CERTIFICATION JOURNEY                       │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. Gap Analysis                                        │
│     └─▶ Assess current state vs. requirements          │
│                                                         │
│  2. Implementation                                      │
│     └─▶ Build and operate the ISMS                     │
│                                                         │
│  3. Internal Audit                                      │
│     └─▶ Verify conformity                              │
│                                                         │
│  4. Management Review                                   │
│     └─▶ Confirm readiness                              │
│                                                         │
│  5. Stage 1 Audit                                       │
│     └─▶ Documentation review                           │
│                                                         │
│  6. Stage 2 Audit                                       │
│     └─▶ Implementation verification                    │
│                                                         │
│  7. Certification                                       │
│     └─▶ Certificate issued (3-year validity)           │
│                                                         │
│  8. Surveillance Audits                                 │
│     └─▶ Annual verification                            │
│                                                         │
│  9. Recertification                                     │
│     └─▶ Every 3 years                                  │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### Maintaining Certification

- Address audit findings promptly
- Conduct regular internal audits
- Hold management reviews
- Maintain documentation
- Continue improvement
- Prepare for surveillance audits

## Maturity Assessment

### ISMS Maturity Levels

| Level | Characteristics |
|-------|-----------------|
| **Initial** | Ad hoc, reactive, individual efforts |
| **Developing** | Basic processes, inconsistent |
| **Defined** | Standardized, organization-wide |
| **Managed** | Measured, controlled, data-driven |
| **Optimizing** | Continuous improvement, leading practice |

### Maturity Indicators

| Area | Mature Practice |
|------|-----------------|
| **Governance** | Security integrated into business |
| **Risk** | Proactive, quantified, business-aligned |
| **Controls** | Automated, continuously monitored |
| **Incidents** | Rapid detection, effective response |
| **Improvement** | Systematic, innovation-driven |

---

## Key Takeaways

1. **Measure what matters** — Meaningful, actionable metrics
2. **Audit regularly** — Independent verification
3. **Management must engage** — Regular reviews, decisions
4. **Address root causes** — Not just symptoms
5. **Continuous improvement** — Never stop enhancing
6. **Certification is ongoing** — Maintain, don't just achieve
